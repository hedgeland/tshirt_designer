Run `git status`, `git diff`, and `git log --oneline -5` in parallel. Review all staged and unstaged changes, then:

1. Stage all modified and new files relevant to the project (exclude `.env`, secrets, and generated output).
2. Write a concise commit message that explains *why* the change was made, not just what changed.
3. Commit and push to origin.
4. Report the commit hash and message when done.
